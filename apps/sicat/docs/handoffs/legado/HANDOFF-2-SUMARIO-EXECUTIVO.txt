📈 DETALHES DOS 7 HEALTH ENDPOINTS

1. /ping (GET)
   └─ Health check simples para load balancers
   └─ Sem autenticação (security: [])
   └─ Retorna: { status: "ok", timestamp }

2. /health/system (GET)
   └─ Status geral da API
   └─ Sem autenticação
   └─ Retorna: { status, version, uptime_ms, environment }

3. /health/workers (GET)
   └─ Heartbeat e estatísticas dos workers
   └─ Sem autenticação
   └─ Retorna: Lista de workers com status e métricas

4. /health/jobs/active (GET)
   └─ Jobs em execução (queued, running, retry_wait)
   └─ Sem autenticação
   └─ Retorna: Contagem por status + job mais antigo

5. /health/jobs/dlq (GET)
   └─ Jobs em dead letter queue
   └─ Sem autenticação
   └─ Retorna: Lista de jobs movidos para DLQ + motivo

6. /health/metrics/performance (GET)
   └─ Percentis de latência (p50, p95, p99)
   └─ Sem autenticação
   └─ Retorna: Métricas de execução e utilização

7. /maintenance/cleanup (POST)
   └─ Disparar limpeza de jobs antigos
   └─ Sem autenticação
   └─ Retorna: Status + contagem de registros deletados

🗄️ FONTE DE VERDADE: BANCO DE DADOS LOCAL

   └─ Métricas agregadas de latência

4. system_events (migration 004)
